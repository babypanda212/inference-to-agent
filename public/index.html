From one inference call to a local agent 

A weekend hackathon. Build-driven, time-boxed, ship something by Sunday night.

Covers: the gap between classical-ML inference and "using the Claude app" — web fundamentals, the LLM stack, agents, RAG, local-first.  ·  Rule: freeCodeCamp + local docs only, runs entirely on Ollama.

THE STACK  —  you have only ever touched the top and (in classical ML) the bottom.

  TOP     Claude app / Pi          a HARNESS (loop + tools + memory).   <- you live here
           |
  MIDDLE  loop · tools · RAG · prompts  the part you build this weekend.    <- never been here
           |
  BOTTOM  one LLM inference call     text in, text out.                  <- never made one

By Sunday night you will have hand-built a tiny harness yourself.
After that, "what is Pi / the Claude app / an agent" stops being abstract.

Rules of the hackathon

  • Build over read. Every block ends in a script that runs, not notes. Read only as far as the next step needs.
  • Local-first. Everything runs on Ollama on your Mac. No cloud keys. The environmental point, made real.
  • Right-size on purpose. Start with the smallest model that works. Feel the speed/quality tradeoff yourself.
  • Log as you go. One line per block: what you built, what surprised you.

What you ship — the capstone

"ask-my-folder": a fully-local document-Q&A agent

Point it at a folder. Ask a question. It retrieves the relevant pieces (RAG), reasons over them with tools in a loop, and answers — grounded in your files, entirely on your laptop. One project that exercises every layer: inference call → calling from code → prompting → structured output → tool use → the loop → RAG → local serving. A cousin of your content-tool's evidence-trail idea.
